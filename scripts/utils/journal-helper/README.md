# Journal Helper

Infrastructure engineering suffers from a persistent friction problem: the gap between doing the work in a terminal and accurately documenting it later. Journal Helper is a custom Bash-based documentation pipeline built to solve this. It ensures that every architectural decision, operational change, and incident response is captured in real-time, treating documentation as an integrated part of the engineering workflow rather than a retroactive chore.

The pipeline automates the entire lifecycle of a journal entry. It scaffolds Markdown files dynamically from templates, prompts for initial context, and then wraps the user's shell in a tracked terminal session using `script(1)`. Throughout the session, the user can inject phase boundaries live using a custom `note()` function. Upon exit, a robust `perl` and `col` parsing pipeline strips ANSI escape sequences, control characters, and terminal noise. The cleaned transcript is then automatically injected between HTML sentinels (``) directly into the working Markdown document. 

The result is a clean, version-controlled artifact that pairs human rationale with an exact, immutable record of the commands executed. It entirely removes the friction of copy-pasting terminal output, ensuring the repository's history remains deeply technical and highly accurate.

## Usage

Start the pipeline by running the entry script:

```bash
./start.sh
```

1. Select whether to create a new entry or edit an existing one.
2. Choose the entry type (e.g., Incident, ADR, Operations).
3. The script will open your $EDITOR to draft the initial context.
4. Close the editor. If the template has terminal tracking sentinels, the script willl drops you into a tracked terminal session.
5.  Do your work. Use the note command to label phases (e.g., note "Phase 1: Firewall rules").
6. Type exit. The transcript is cleaned, injected into your document, and the editor reopens for final review.

Documentation becomes easy when you can use the `note` command to track what you are doing in real-time, with minimal friction, and in-context. Your main task becomes editing, instead of reconstructing from memory.
